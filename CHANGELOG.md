# 更新日志

## 3.0.0 - 2026-03-29

对比上游原仓库 `MoePlayer/APlayer-Typecho` 的正式发布版 `v2.1.2`。

### 新增

- 新增完整 `kuwo` 平台支持，覆盖短代码解析、后端解析、播放链接、封面、歌词与自动识别。
- 新增 Typecho 1.3 兼容支持，适配命名空间类名与双内容钩子注册。
- 新增结构化 API 响应兜底与调试日志，覆盖无效 JSON、空歌单、鉴权失败和 referer 拦截等场景。
- 新增仓库内置 `CHANGELOG.md`，用于后续版本维护。

### 变更

- 插件版本提升至 `3.0.0`。
- 跟随上游 Meting 的平台与解析逻辑更新，同步调整插件内置解析实现。
- 插件 PHP 入口重构为 Typecho 1.3 风格的命名空间写法，同时保持原有短代码语法与 `action/metingapi` 接口结构不变。
- 插件支持的平台集合调整为：`netease`、`tencent`、`kugou`、`baidu`、`kuwo`。
- 升级前端播放器脚本，并同步前后端的平台识别与解析规则。
- 仓库链接、发布页入口和插件自更新检查地址切换到 `https://github.com/mikusaa/Typecho-Plugin-APlayer`。
- 插件配置保存时不再反复重置 `salt`，避免已有页面中的 `auth` 参数失效。

### 修复

- 修复 Typecho 1.3 下播放器无法正常渲染的问题。
- 修复同源 referer 校验未正确处理端口信息的问题，保证部署在非默认端口上的站点也能正常请求播放器接口。
- 修复 API 输出处理，在列表接口中统一返回合法 JSON 数组，并为链接、歌词、封面接口增加更稳妥的兜底逻辑。
- 修复 MySQL 与 SQLite 缓存驱动在 Typecho 1.3 下的类名兼容问题。
- 修复 MetingJS 前端错误处理，避免 JSON 解析失败或空歌单时页面静默空白。

### 移除

- 移除 `xiami` 平台在插件后端、短代码解析与自动识别中的全部支持逻辑。
- 移除已不属于当前 Meting 上游平台集合的旧兼容分支。

### 说明

- 本版本明确将 `xiami` 视为已废弃平台，不再为旧文章中的 `server="xiami"` 提供兼容层。
- 个别平台的接口可用性仍受上游音乐平台本身影响；本次发布的重点是完成 Typecho 1.3 兼容适配、统一平台支持范围，并提升播放器加载稳定性。
